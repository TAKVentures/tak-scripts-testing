## 1. Subject Line Best Practices

### Length

Keep subject lines between **40–60 characters**. Most desktop clients display up to ~60 characters; mobile clients display 30–40. Shorter is almost always better — truncation mid-word is worse than a well-edited short line.

| Context | Target Length | Example |
|---|---|---|
| Alert | 35–45 chars | `Action Required: Low Inventory — Widget A` |
| Digest | 40–55 chars | `Your Weekly Summary — March 23, 2026` |
| Invoice | 45–60 chars | `Invoice #1042 from TAK Scripts — $450.00 Due` |
| Follow-up | 40–55 chars | `Following Up: Proposal Sent March 20` |
| Notification | 30–45 chars | `Order #8821 Has Shipped` |

### Emoji Use

Use one emoji maximum, placed at the start or end of the subject. Emoji in the middle of text can break rendering in some clients.

```
✅ Task Complete — Report Ready for Review
🔴 Action Required: Payment Overdue
📋 Weekly Digest — 5 Items Need Your Attention
```

Rules:
- Use emoji to signal status at a glance (red = urgent, green = complete)
- Test that the emoji renders on both Android Gmail and iOS Mail
- Never use decorative emoji that add no meaning
- Avoid strings of multiple emoji (looks like spam)

### Personalization

Including the recipient's name or a relevant identifier increases open rates. GmailApp makes this easy since you are constructing the email per-recipient in script.

```
Invoice #1042 for Acme Corp — Due April 1
Your March 23 Run Report, John
Alert for Store #4: Inventory Below Threshold
```

### Urgency Signals

Convey urgency through specificity, not hype. Specific deadlines outperform vague superlatives.

```
// Good — specific and factual
"Payment Due Today: Invoice #1042"
"Inventory Alert: Reorder Needed by Friday"

// Bad — hype language that triggers spam filters
"URGENT!!! Act Now — Critical Alert!!!"
"You MUST respond immediately"
```

### Avoiding Spam Triggers

Words and patterns that increase spam scoring:

- All-caps words: `FREE`, `URGENT`, `ACT NOW`
- Excessive punctuation: `!!!`, `???`
- Dollar signs in subject: `$$$ Save money $$$`
- Misleading RE: or FWD: prefixes on non-replies
- Phrases: "no obligation", "winner", "limited time offer", "click here"

For internal business emails and transactional emails, your domain reputation carries more weight than trigger words — but clean subject lines still matter for deliverability and tone.

## 2. Email Structure

Every email should follow this vertical stack. Each section is a separate table row.

```
┌─────────────────────────────┐
│  Preheader (hidden text)    │  ← Appears in inbox preview
├─────────────────────────────┤
│  Header / Logo              │  ← Brand anchor, ~80px tall
├─────────────────────────────┤
│  Body                       │  ← Main content
│    Title / Headline         │
│    Supporting text          │
│    Data / Table             │
│    CTA Button               │
├─────────────────────────────┤
│  Footer                     │  ← Legal, sender info, unsubscribe
└─────────────────────────────┘
```

### Preheader Text

The preheader is hidden text that follows the subject line in the inbox preview pane. Without it, email clients pull the first visible text from the body (often "View in browser" or table headers — both bad).

```html
<!-- Place immediately after <body> tag -->
<div style="display:none;max-height:0;overflow:hidden;mso-hide:all;font-size:1px;color:#ffffff;line-height:1px;">
  3 items require your attention this week. Sales are up 12% month-over-month.
</div>
```

Keep preheader to 85–100 characters. It supplements the subject line — do not repeat it.

### Header

The header establishes brand identity. Keep it simple: company name in text or a small logo image (with alt text).

```html
<tr>
  <td style="background-color:#1a1a2e;padding:24px 32px;text-align:center;">
    <span style="font-family:Arial,sans-serif;font-size:20px;font-weight:bold;color:#ffffff;letter-spacing:1px;">
      TAK Scripts
    </span>
  </td>
</tr>
```

### Body

Lead with a single clear headline, followed by supporting text. Put the most important information above the fold (within the first ~300px of content). One topic per email whenever possible.

### CTA Button

One primary CTA per email. Place it after the core message, before supporting detail. See Section 6 for full button styling.

### Footer

Mandatory. See Section 12.

## 3. HTML Email Layout

### Why Table-Based Layout

Email clients — especially Outlook — do not support CSS `flexbox`, `grid`, or `position`. The only reliable layout primitive across all clients (Gmail, Outlook, Apple Mail, Yahoo) is HTML tables. This is not optional.

### Base Template Structure

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Email Title</title>
</head>
<body style="margin:0;padding:0;background-color:#f4f4f4;font-family:Arial,sans-serif;">

  <!-- Preheader -->
  <div style="display:none;max-height:0;overflow:hidden;mso-hide:all;font-size:1px;color:#f4f4f4;line-height:1px;">
    Preview text goes here — 85 to 100 characters describing the email content.
  </div>

  <!-- Outer wrapper -->
  <table width="100%" cellpadding="0" cellspacing="0" border="0" style="background-color:#f4f4f4;">
    <tr>
      <td align="center" style="padding:24px 16px;">

        <!-- Content container — max 600px -->
        <table width="100%" cellpadding="0" cellspacing="0" border="0"
               style="max-width:600px;background-color:#ffffff;border-radius:8px;overflow:hidden;box-shadow:0 2px 8px rgba(0,0,0,0.08);">

          <!-- HEADER ROW -->
          <tr>
            <td style="background-color:#1a1a2e;padding:24px 32px;text-align:center;">
              <span style="font-family:Arial,sans-serif;font-size:20px;font-weight:bold;color:#ffffff;">
                TAK Scripts
              </span>
            </td>
          </tr>

          <!-- BODY ROW -->
          <tr>
            <td style="padding:32px;">
              <!-- Content here -->
            </td>
          </tr>

          <!-- FOOTER ROW -->
          <tr>
            <td style="background-color:#f8f8f8;border-top:1px solid #e5e5e5;padding:24px 32px;text-align:center;">
              <!-- Footer content here -->
            </td>
          </tr>

        </table>
      </td>
    </tr>
  </table>

</body>
</html>
```

### Inline CSS Rules

Gmail strips `<style>` blocks in some rendering contexts (especially forwarded emails and third-party clients). Always inline critical styles directly on elements.

- Every `style` attribute should be self-contained
- Do not rely on class selectors or external stylesheets
- Use `cellpadding="0"` and `cellspacing="0"` on all tables — then control spacing via `padding` in `style`
- Use `border="0"` on all tables to remove default borders
- Set `width` as HTML attributes on `<td>` for Outlook compatibility, not just CSS

### Max-Width 600px

The 600px container is a hard standard for email. Content wider than 600px will be clipped or force horizontal scrolling on mobile. The outer wrapper table is `width="100%"` to fill the viewport; the inner container is `max-width:600px`.

## 4. Typography in Email

### Web-Safe Font Stack

Do not use Google Fonts or custom web fonts — they are not reliably supported in Gmail or Outlook. Stick to system fonts:

| Font | Style | Use Case |
|---|---|---|
| Arial, Helvetica, sans-serif | Clean, modern | Body text, UI labels |
| Georgia, serif | Authoritative | Headlines, quotes |
| Trebuchet MS, sans-serif | Friendly | Subheadings, alerts |
| Courier New, monospace | Technical | IDs, codes, order numbers |

### Font Size Scale

```
Headline:       24px, font-weight: bold
Subheadline:    18px, font-weight: bold
Body text:      15px, font-weight: normal
Small / label:  13px, font-weight: normal
Footer text:    11px, font-weight: normal
```

Never go below **11px** — Gmail will upscale text smaller than 13px on mobile, causing layout shifts.

### Line Height

```css
Body text:      line-height: 1.6
Headlines:      line-height: 1.2
Table cells:    line-height: 1.4
```

Always set `line-height` as a unitless multiplier (not `px`) so it scales properly with font size.

### Paragraph Spacing

Use `margin-bottom` on `<p>` tags or separate `<tr>` rows for spacing. Do not use `<br><br>` for paragraph breaks — it is unreliable across clients.

```html
<p style="font-family:Arial,sans-serif;font-size:15px;line-height:1.6;color:#333333;margin:0 0 16px 0;">
  Your weekly summary is ready. Here is what happened this week.
</p>
```

## 5. Color in Email

### Core Palette

Always specify colors as 6-digit hex codes. Do not use CSS named colors (`red`, `blue`) or `rgb()` — Outlook sometimes ignores them.

```
Background (page):     #f4f4f4  — Light gray, keeps the container visually separated
Background (card):     #ffffff  — White, the main email body
Header background:     #1a1a2e  — Deep navy, strong brand anchor
Header text:           #ffffff  — White

Body text (primary):   #1a1a1a  — Near-black, high contrast
Body text (secondary): #555555  — Medium gray, supporting text
Body text (muted):     #888888  — Light gray, labels, timestamps

Divider / border:      #e5e5e5  — Subtle separator lines
Highlight background:  #f0f4ff  — Light blue tint, for callout boxes

Footer background:     #f8f8f8  — Off-white
Footer text:           #999999  — Muted gray
```

### Button Colors

```
Primary CTA:    #2563eb  (blue)   — Main action
Success:        #16a34a  (green)  — Confirm, approve, complete
Warning:        #d97706  (amber)  — Review, caution
Danger:         #dc2626  (red)    — Alert, urgent, overdue
Neutral:        #6b7280  (gray)   — Secondary action, cancel
```

### Status Colors

For status badges and row highlights in tables:

```
Status: Success
  Background: #f0fdf4   Border: #bbf7d0   Text: #15803d

Status: Warning
  Background: #fffbeb   Border: #fde68a   Text: #92400e

Status: Error / Critical
  Background: #fef2f2   Border: #fecaca   Text: #991b1b

Status: Info / Neutral
  Background: #eff6ff   Border: #bfdbfe   Text: #1d4ed8

Status: Inactive / Pending
  Background: #f9fafb   Border: #e5e7eb   Text: #6b7280
```

### Text Contrast

Never put light gray text on a white background for body content. The WCAG AA minimum contrast ratio is 4.5:1 for normal text. Use `#555555` or darker for any text that conveys meaning.

## 6. Call-to-Action Buttons

### The Table-Based Button Pattern

CSS-only buttons break in Outlook. For broad compatibility, use a table-based button — a `<td>` styled as a button rather than an `<a>` with `display:block`.

```html
<!-- Single CTA button -->
<table cellpadding="0" cellspacing="0" border="0" style="margin:24px 0;">
  <tr>
    <td align="center" style="background-color:#2563eb;border-radius:6px;">
      <a href="https://your-link.com"
         style="display:inline-block;font-family:Arial,sans-serif;font-size:15px;font-weight:bold;
                color:#ffffff;text-decoration:none;padding:14px 28px;letter-spacing:0.3px;">
        View Report
      </a>
    </td>
  </tr>
</table>
```

### Button Sizing

```
Padding:       14px top/bottom, 28px left/right (minimum)
Font size:     15px (never smaller than 14px for touch targets)
Border-radius: 6px (professional), 4px (conservative/Outlook-safe)
Min-width:     160px if button text is short
```

### Button Color Variants

```html
<!-- Success / Confirm -->
<td style="background-color:#16a34a;border-radius:6px;">
  <a href="#" style="display:inline-block;font-family:Arial,sans-serif;font-size:15px;
                     font-weight:bold;color:#ffffff;text-decoration:none;padding:14px 28px;">
    Approve
  </a>
</td>

<!-- Warning / Review -->
<td style="background-color:#d97706;border-radius:6px;">
  <a href="#" style="display:inline-block;font-family:Arial,sans-serif;font-size:15px;
                     font-weight:bold;color:#ffffff;text-decoration:none;padding:14px 28px;">
    Review Now
  </a>
</td>

<!-- Danger / Urgent -->
<td style="background-color:#dc2626;border-radius:6px;">
  <a href="#" style="display:inline-block;font-family:Arial,sans-serif;font-size:15px;
                     font-weight:bold;color:#ffffff;text-decoration:none;padding:14px 28px;">
    Take Action
  </a>
</td>

<!-- Ghost / secondary (use sparingly) -->
<td style="background-color:#ffffff;border-radius:6px;border:2px solid #2563eb;">
  <a href="#" style="display:inline-block;font-family:Arial,sans-serif;font-size:15px;
                     font-weight:bold;color:#2563eb;text-decoration:none;padding:12px 26px;">
    View Details
  </a>
</td>
```

### One Primary CTA Rule

Every email should have **one** primary action. If you need a secondary link (e.g., "View in browser" or "Download PDF"), place it as a plain text link below the button, not as a second button.

```html
<p style="font-family:Arial,sans-serif;font-size:13px;color:#888888;text-align:center;margin:8px 0 0 0;">
  Or <a href="https://your-link.com" style="color:#2563eb;text-decoration:underline;">open in browser</a>
</p>
```

## 7. Alert and Notification Emails

Alerts need to communicate status at a glance. Lead with the status badge, then the headline, then detail.

### Status Badge

```html
<!-- Red / Critical -->
<div style="display:inline-block;background-color:#fef2f2;border:1px solid #fecaca;
            border-radius:4px;padding:4px 12px;margin-bottom:16px;">
  <span style="font-family:Arial,sans-serif;font-size:12px;font-weight:bold;color:#991b1b;
               text-transform:uppercase;letter-spacing:0.5px;">
    &#9679; Critical Alert
  </span>
</div>

<!-- Yellow / Warning -->
<div style="display:inline-block;background-color:#fffbeb;border:1px solid #fde68a;
            border-radius:4px;padding:4px 12px;margin-bottom:16px;">
  <span style="font-family:Arial,sans-serif;font-size:12px;font-weight:bold;color:#92400e;
               text-transform:uppercase;letter-spacing:0.5px;">
    &#9679; Needs Attention
  </span>
</div>

<!-- Green / Resolved -->
<div style="display:inline-block;background-color:#f0fdf4;border:1px solid #bbf7d0;
            border-radius:4px;padding:4px 12px;margin-bottom:16px;">
  <span style="font-family:Arial,sans-serif;font-size:12px;font-weight:bold;color:#15803d;
               text-transform:uppercase;letter-spacing:0.5px;">
    &#10003; Resolved
  </span>
</div>
```

### Emoji as Icons

Emoji render consistently in Gmail and Apple Mail and are preferable to image icons (which can be blocked). Use them for quick visual anchoring:

```
🔴  Critical / Error / Stop
🟡  Warning / Caution / Pending
🟢  Success / Complete / Active
📋  Report / Summary / List
📦  Inventory / Package / Order
💰  Payment / Invoice / Revenue
📧  Email / Message / Notification
⏰  Time-sensitive / Reminder / Due
🔗  Link / Reference / View
```

### Alert Summary Table

For alerts that contain multiple data points:

```html
<table width="100%" cellpadding="0" cellspacing="0" border="0"
       style="border:1px solid #e5e5e5;border-radius:6px;overflow:hidden;margin:16px 0;">
  <!-- Column headers -->
  <tr style="background-color:#f8f8f8;">
    <td style="font-family:Arial,sans-serif;font-size:12px;font-weight:bold;color:#555555;
               padding:10px 16px;text-transform:uppercase;letter-spacing:0.5px;border-bottom:1px solid #e5e5e5;">
      Field
    </td>
    <td style="font-family:Arial,sans-serif;font-size:12px;font-weight:bold;color:#555555;
               padding:10px 16px;text-transform:uppercase;letter-spacing:0.5px;border-bottom:1px solid #e5e5e5;">
      Value
    </td>
  </tr>
  <!-- Data row -->
  <tr>
    <td style="font-family:Arial,sans-serif;font-size:14px;color:#555555;padding:12px 16px;border-bottom:1px solid #f0f0f0;">
      Triggered At
    </td>
    <td style="font-family:Arial,sans-serif;font-size:14px;color:#1a1a1a;font-weight:bold;padding:12px 16px;border-bottom:1px solid #f0f0f0;">
      March 23, 2026 at 9:14 AM
    </td>
  </tr>
  <tr>
    <td style="font-family:Arial,sans-serif;font-size:14px;color:#555555;padding:12px 16px;">
      Current Value
    </td>
    <td style="font-family:Arial,sans-serif;font-size:14px;color:#dc2626;font-weight:bold;padding:12px 16px;">
      4 units (threshold: 10)
    </td>
  </tr>
</table>
```

### Alert Callout Box

For a single prominent message (e.g., one critical value):

```html
<div style="background-color:#fef2f2;border-left:4px solid #dc2626;border-radius:4px;
            padding:16px 20px;margin:16px 0;">
  <p style="font-family:Arial,sans-serif;font-size:14px;font-weight:bold;color:#991b1b;margin:0 0 4px 0;">

  </p>

    Widget A — Model XR4 is at 4 units. Minimum threshold is 10. Reorder immediately.
  </p>
</div>
```

## 8. Digest Emails

Digest emails consolidate multiple updates into one message. They are longer than alerts and need clear visual hierarchy.

### Structure for Digest Emails

```
1. Summary stats bar (3–4 KPIs at a glance)
2. Section 1 headline + items
3. Section 2 headline + items
4. "View All" link for each section
5. Footer
```

### Summary Stats Bar

```html
<table width="100%" cellpadding="0" cellspacing="0" border="0"
       style="background-color:#f0f4ff;border-radius:6px;margin-bottom:24px;">
  <tr>
    <td width="33%" style="text-align:center;padding:16px 8px;border-right:1px solid #dbeafe;">
      <div style="font-family:Arial,sans-serif;font-size:26px;font-weight:bold;color:#1a1a1a;">
        142
      </div>
      <div style="font-family:Arial,sans-serif;font-size:12px;color:#555555;margin-top:4px;">
        Total Orders
      </div>
    </td>
    <td width="33%" style="text-align:center;padding:16px 8px;border-right:1px solid #dbeafe;">
      <div style="font-family:Arial,sans-serif;font-size:26px;font-weight:bold;color:#16a34a;">
        $8,420
      </div>
      <div style="font-family:Arial,sans-serif;font-size:12px;color:#555555;margin-top:4px;">
        Revenue
      </div>
    </td>
    <td width="33%" style="text-align:center;padding:16px 8px;">
      <div style="font-family:Arial,sans-serif;font-size:26px;font-weight:bold;color:#dc2626;">
        3
      </div>
      <div style="font-family:Arial,sans-serif;font-size:12px;color:#555555;margin-top:4px;">
        Items Need Attention
      </div>
    </td>
  </tr>

```

### Section with Itemized List

```html
<!-- Section header -->
<p style="font-family:Arial,sans-serif;font-size:16px;font-weight:bold;color:#1a1a1a;
           margin:0 0 4px 0;border-bottom:2px solid #2563eb;padding-bottom:8px;">
  📦 Low Inventory Items
</p>

<!-- Item row (repeat for each item) -->
<table width="100%" cellpadding="0" cellspacing="0" border="0" style="margin-bottom:2px;">
  <tr>
    <td style="font-family:Arial,sans-serif;font-size:14px;color:#1a1a1a;padding:10px 0;
               border-bottom:1px solid #f0f0f0;">
      Widget A — Model XR4
    </td>
    <td align="right" style="font-family:Arial,sans-serif;font-size:14px;color:#dc2626;
                              font-weight:bold;padding:10px 0;border-bottom:1px solid #f0f0f0;
                              white-space:nowrap;">
      4 units left
    </td>
  </tr>
</table>

<!-- View All link -->
<p style="font-family:Arial,sans-serif;font-size:13px;text-align:right;margin:8px 0 24px 0;">
  <a href="https://your-link.com/inventory" style="color:#2563eb;text-decoration:none;font-weight:bold;">
    View all inventory →
  </a>
</p>
```

### Section Divider

```html
<table width="100%" cellpadding="0" cellspacing="0" border="0">
  <tr>
    <td style="padding:0 0 24px 0;">
      <hr style="border:none;border-top:1px solid #e5e5e5;margin:0;">
    </td>
  </tr>
</table>
```

### Digest Length Rule

Truncate each section to a maximum of **10 items** in the email. Link to a full view for anything beyond that. A digest email that renders 80 rows of data will be clipped by Gmail (102KB limit) and is unreadable regardless.

## 9. Invoice Emails

Invoice emails must be clear, professional, and include every piece of information a recipient needs to pay without following up.

### Invoice Email Structure

```
1. Header with company name
2. Invoice number, date, due date — prominently displayed
3. Billed To section
4. Line items table
5. Totals section (subtotal, tax, total due)
6. Payment instructions / CTA
7. Notes or accepted payment methods
8. Footer
```

<table width="100%" cellpadding="0" cellspacing="0" border="0" style="margin-bottom:24px;">

    </td>
    <td align="right" style="vertical-align:top;">

        Due Date

## 10. Plain Text Fallback

### When to Include

Always include a plain text body alongside the HTML body. It is required for:
- Anti-spam compliance (some filters penalize HTML-only emails)
- Accessibility (screen readers on some mail clients prefer it)
- Email clients with HTML disabled (rare but real in some enterprise environments)

The `GmailApp.sendEmail()` and `MailApp.sendEmail()` functions accept both `body` (plain text) and `htmlBody` parameters. Both should always be populated.

### Plain Text Formatting Conventions

```
SUBJECT: Invoice #1042 from TAK Scripts — $450.00 Due

Hi [Name],

Your invoice is ready.

INVOICE DETAILS
---------------
Invoice Number: #1042
Issued:         March 23, 2026
Due Date:       April 1, 2026

ITEMS
-----
Monthly Automation Service x1        $400.00
Tax (12.5%)                           $50.00
                                  ---------
Total Due:                           $450.00

PAY ONLINE
----------
https://your-payment-link.com/inv/1042

PAYMENT BY BANK TRANSFER
BSB: 123-456 | Account: 78901234
Reference: INV-1042

Questions? Reply to this email.

---
TAK Scripts | your-domain.com
This is an automated message. Contact support@your-domain.com for help.
```

Plain text rules:
- Use ALL CAPS for section headers
- Use dashes (`---`) as dividers
- Include the full URL, never a hyperlinked anchor phrase
- Keep lines under 80 characters
- Right-align numeric columns with spaces for readability

## 11. Responsive Email

### Core Responsive Approach

Email CSS has no `@media` query support in Gmail's webmail client. Design for mobile-first within a 600px container — then you do not need media queries to get acceptable mobile rendering.

Key rules that make emails "responsive enough" without media queries:

```
- Container max-width:600px on outer table
- Use percentage widths for fluid columns: width="50%"
- Never use fixed pixel widths on inner content elements
- Images: width="100%" style="max-width:100%;display:block;"
- Font sizes no smaller than 14px (Gmail upscales below 13px anyway)
- Button padding generous enough for touch: min 14px vertical
```

### Fluid Images

```html
<img src="https://your-image.com/image.png"
     alt="Descriptive alt text here"
     width="600"
     style="max-width:100%;height:auto;display:block;border:0;">
```

- Always set `width` as an HTML attribute (Outlook uses it for layout)
- Always set `max-width:100%` in style (mobile clients use it)
- Always set `display:block` to prevent phantom gaps below images
- Always include `alt` text — images may be blocked by default

### Fluid Two-Column Layout

```html
<table width="100%" cellpadding="0" cellspacing="0" border="0">
  <tr>
    <td width="50%" valign="top" style="padding:0 8px 0 0;">
      <!-- Left column content -->
    </td>
    <td width="50%" valign="top" style="padding:0 0 0 8px;">
      <!-- Right column content -->
    </td>
  </tr>
</table>
```

For guaranteed mobile compatibility, prefer single-column layout for all GmailApp-generated emails. Two-column layouts do not stack automatically without `@media` queries.

### Mobile Font Sizes

Since `@media` queries do not work in Gmail webmail, pick base sizes that work on both:

```
Headlines:   22px minimum — readable on mobile, not oversized on desktop
Body text:   15px — comfortable on mobile, clean on desktop
Labels:      13px — smallest usable size on mobile
```

## 13. What to Avoid

### Image-Only Emails

Never send an email where all meaningful content is inside images. Images are blocked by default in many clients (Outlook, corporate environments). If your email is one large image with text baked in, recipients see a blank email.

- Always use HTML text for all critical content
- Images are decorative only; the email must make sense with images turned off
- Every image must have descriptive `alt` text

### Too Many Links

More than 5–6 links in one email increases spam score and reduces click-through on any individual link. For digest emails with many links, use a single "View Full Report" CTA and surface only the 2–3 most important direct links inline.

### Spam Trigger Patterns

```
Avoid in subject and body:
- "Click here" — use descriptive anchor text instead
- "Free", "guaranteed", "no cost"
- "You've been selected", "Congratulations"
- Excessive exclamation marks: "Great news!!!"
- ALL CAPS words outside of intentional label contexts
- Mismatched link text and destination (link says "Report" but URL goes elsewhere)
- Very short email body with one big image and one link
```

### Missing Alt Text

Every `<img>` tag must have an `alt` attribute, even if the value is empty for purely decorative images (`alt=""`). Missing `alt` attributes reduce accessibility and can lower deliverability scores.

### Broken Layout Patterns

```
- Never use <div> for layout structure — use <table>
- Never use CSS float or flexbox for positioning
- Never embed base64-encoded images inline — use hosted URLs
- Never use JavaScript in email — it is stripped by every client
- Never use <form> or <input> elements — stripped by Gmail
- Never use external stylesheets or <style> blocks for critical styles
```

### Oversized Emails

Keep total HTML size under **100KB**. Gmail clips emails over 102KB and shows "Message clipped — view entire message", which breaks the email experience and can hide the footer and unsubscribe link.

For digest emails with many data rows, truncate to the top 10–15 items and link to the full report externally.

## 14. GmailApp / MailApp Implementation

### MailApp vs GmailApp

| Feature | MailApp | GmailApp |
|---|---|---|
| Authorization scope required | Simple (mail) | Full Gmail scope |
| Daily quota — consumer account | 100/day | 100/day |
| Daily quota — Google Workspace | 1,500/day | 1,500/day |
| Send as alias | No | Yes, with `from` param |
| Attach Drive files | Yes | Yes |
| Access sent mail / threads | No | Yes |
| Best for | Simple alerts, notifications | Full email control, alias sending |

Use `MailApp` for simple alerting scripts. Use `GmailApp` when you need to send as a different address, manage threads, or inspect sent mail.

### sendEmail() Parameter Reference

Both services share the same parameter object structure:

```javascript
MailApp.sendEmail({
  to: 'recipient@example.com',
  cc: 'cc@example.com',                          // optional
  bcc: 'bcc@example.com',                        // optional
  subject: 'Invoice #1042 — $450.00 Due',
  body: plainTextBody,                            // plain text fallback — always provide
  htmlBody: htmlBody,                             // HTML version
  name: 'TAK Scripts',                           // display name for sender
  replyTo: 'support@your-domain.com',            // optional reply-to address
  attachments: [file.getAs('application/pdf')]   // optional
});
```

### Code Pattern: Send with Quota Guard

```javascript
function sendAlert(data) {
  const htmlBody  = buildAlertEmail(data);
  const plainBody = buildAlertPlainText(data);

  MailApp.sendEmail({
    to:       data.recipientEmail,
    subject:  data.subject,
    body:     plainBody,
    htmlBody: htmlBody,
    name:     'TAK Scripts Alerts'
  });
}

function sendBatchSafe(emailList, buildFn) {
  let remaining = MailApp.getRemainingDailyQuota();

  for (let i = 0; i < emailList.length; i++) {
    if (remaining <= 5) {
      console.warn(`Quota nearly exhausted. Stopped at item ${i} of ${emailList.length}.`);
      break;
    }
    MailApp.sendEmail(buildFn(emailList[i]));
    remaining--;
    Utilities.sleep(200); // avoid burst rate limiting
  }
}
```

### Quota Limits Reference

```
Consumer Google account:   100 emails/day
Google Workspace account:  1,500 emails/day
Quota resets:              Midnight Pacific Time

Check remaining quota:
  MailApp.getRemainingDailyQuota()  // returns integer

GmailApp does not expose a getRemainingDailyQuota() method — use MailApp to check.
```

### Pre-Send Checklist

```
[ ] Subject line is 40–60 chars with no spam trigger words
[ ] Preheader text is set and unique from the subject line
[ ] HTML body uses table-based layout — no <div> for structure
[ ] All CSS is inlined on each element — no <style> block for critical styles
[ ] Plain text body (body:) is populated alongside htmlBody:
[ ] All URLs are absolute (https://), not relative paths
[ ] Images have alt text; email reads correctly with images disabled
[ ] Total HTML size is under 100KB
[ ] Footer includes sender info and how to stop receiving emails
[ ] Quota checked before any batch send loop
[ ] Tested by sending to a real Gmail inbox before deploying
```
